## Requisitos funcionais

O sistema deve permitir:

- Cadastro, listagem e exclusão de pessoas;
- Exclusão automática das transações relacionadas ao excluir uma pessoa;
- Cadastro e listagem de transações;
- Validação de regras de negócio para cadastro de transações;
- Consulta de totais de receitas, despesas e saldo por pessoa;
- Consulta dos totais gerais.

## Requisitos não funcionais

O projeto deve utilizar:

- .NET com C# no backend;
- React com TypeScript no frontend;
- PostgreSQL como banco de dados;
- Entity Framework Core com migrations para persistência;
